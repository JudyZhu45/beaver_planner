# AI Chat Feature — 完整测试计划

> **目的**：验证 AI 能够正确执行所有 CRUD 操作、识别时间冲突、正确路由 Mode 1（直接执行）/ Mode 2（确认后执行），以及撤销功能。
>
> **测试日期基准**：以实际测试当天为 `TODAY`，测试前请记录当天日期。
>
> **测试顺序**：按章节顺序执行，每个用例独立记录结果。

---

## 测试前准备

1. 打开 app，进入 AI Chat 页面
2. 确保任务列表**干净**（或记录已有任务，避免干扰）
3. 如有需要，先手动创建"种子任务"（部分测试用例依赖预先存在的任务）

## 关键验收标准

| 功能 | 标准 |
|------|------|
| Mode 1 触发条件 | 单任务 + 明确时间 + **无冲突** → 直接执行 |
| Mode 2 触发条件 | 批量 / 模糊时间 / AI 自主选时 / **任何冲突** → 必须弹确认 |
| 冲突检测 | 哪怕 1 分钟重叠也必须识别，不得静默写入 |
| 原子写入 | 冲突解决时，新任务 + 被移动任务必须在同一次回复中全部输出 |
| 撤销 | 所有操作类型（创建/更新/删除/完成）均可一键还原 |
| 客户端校验 | 非法 UUID / 时间顺序错误 / 空标题 → 操作被拦截，显示 warning toast |
